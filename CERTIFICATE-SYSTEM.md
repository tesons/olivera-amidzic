# Аутоматски систем за приказ сертификата

## Преглед

Овај систем аутоматски приказује све PDF и слике из фолдера `/assets/certificates/` на страници `/pages/sertifikati` без потребе за било каквим скриптовима или претпроцесирањем.

## Како функционише

Систем користи Jekyll-ов уграђени `site.static_files` механизам који аутоматски детектује све статичке фајлове у пројекту. На овај начин:

1. **Аутоматска детекција**: Jekyll аутоматски скенира фолдер `assets/certificates/` при сваком build-у
2. **Филтрирање**: Приказују се само PDF, PNG, JPG и JPEG фајлови  
3. **Паметно именовање**: Назив фајла се аутоматски преводи у читљив наслов
4. **PDF превју**: Користи се PDF.js за генерисање слика прве стране PDF докумената
5. **Респонзиван дизајн**: Галерија се аутоматски прилагођава величини екрана

## Како додати нови сертификат

**Једноставно копирање фајла:**

1. Додајте PDF или слику у фолдер `assets/certificates/`
2. Commit и push промена у Git
3. Готово! Фајл ће се аутоматски појавити на страници

**Нема потребе за:**
- Покретање скриптова
- Ажурирање конфигурационих фајлова
- Мануално додавање ставки у листе
- Било какво претпроцесирање

## Подржани формати

- **PDF фајлови**: Аутоматско генерисање превју слике + отварање у новом табу
- **PNG, JPG, JPEG слике**: Директно приказивање као слике

## Именовање фајлова

Систем користи назив фајла (без екстензије) као наслов - без икакве обраде или претварања:

- `SKM_308e22042514310.pdf` → "SKM_308e22042514310"
- `sertifikat-1784_2019-10939.pdf` → "sertifikat-1784_2019-10939"  
- `BIGZ_potvrda_vebinar_hemije.pdf` → "BIGZ_potvrda_vebinar_hemije"
- `Olivera Amidžić.pdf` → "Olivera Amidžić"
- `My Certificate 2024.jpg` → "My Certificate 2024"

За читљиве наслове дајте фајловима описне називе при додавању.

## Технички детаљи

### Архитектура
- **Jekyll Liquid темплејти**: Обрађују фајлове током build времена
- **PDF.js библиотека**: Генерише превју слике за PDF документе у браузеру
- **CSS Grid**: Респонзиван layout галерије
- **JavaScript**: Отварање докумената у новом табу

### Компатибилност
- ✅ GitHub Pages (без custom plugin-ова)
- ✅ Локални Jekyll развој
- ✅ Било који Jekyll хостинг
- ✅ Статичка дистрибуција

### Перформансе
- Нема server-side обраде
- PDF превју се генерише асинхроно у браузеру
- `loading="lazy"` за слике
- Минималан JavaScript footprint

## Deployment

### GitHub Pages
1. Додајте фајлове у `assets/certificates/`
2. Commit и push
3. GitHub Pages ће аутоматски rebuild сајт
4. Нови фајлови се појављују аутоматски

### Локални развој
```bash
# Docker приступ
docker run --rm -v "$(pwd):/srv/jekyll" jekyll/jekyll:latest jekyll serve

# Или нативни Jekyll (ако је инсталиран)
bundle exec jekyll serve
```

## Troubleshooting

**Нови фајл се не појављује:**
- Проверите да ли је фајл у `assets/certificates/` фолдеру
- Проверите формат (PDF, PNG, JPG, JPEG)
- Rebuild Jekyll сајт

**PDF превју се не учитава:**
- Систем има fallback на PDF иконицу
- Проверите да ли је PDF фајл валидан
- PDF.js се учитава из CDN-а (потребна интернет веза)
